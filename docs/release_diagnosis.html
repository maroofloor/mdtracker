🚨 배포 블로커 (반드시 수정 후 배포)

이 항목들은 일반 유저가 앱을 정상적으로 사용하지 못하게 막는 문제입니다. 배포 전에 전부 해결해야 합니다.

BLOCKER Tesseract 설치 장벽 — 일반 유저 진입 불가
OCR의 핵심 의존성인 tesseract를 유저가 직접 설치해야 하고, ocr_config.json에 경로까지 수동 입력해야 합니다. 일반 유저는 이 과정에서 100% 이탈합니다.

해결 방향 ① (권장): PyInstaller 번들에 Tesseract 바이너리(+ kor 학습 데이터)를 포함시켜 자체 내장.
해결 방향 ②: 앱 첫 실행 시 설치 위자드 팝업 → Tesseract 다운로드 링크 + 경로 자동 감지 버튼 제공.
BLOCKER ocr_config.json — GUI가 없는 설정 파일 노출
tesseract 경로, 모니터 번호, 피드백 URL 등 운영 설정이 JSON 파일에 있습니다. 일반 유저는 JSON을 편집하지 않습니다.

해결 방향: 설정 뷰에 "OCR 설정" 섹션 추가 — Tesseract 경로 찾기 버튼, 모니터 선택 드롭다운, 연결 테스트 버튼.
BLOCKER 3:2 비율 강제 — 일부 화면 크기에서 UI 깨짐
_ASPECT_WIDTH=3, _ASPECT_HEIGHT=2로 창 비율이 3:2로 고정됩니다. 16:9(대다수 모니터), 16:10 노트북에서 창을 늘려도 비율이 3:2로 교정되며, 최대화 시 화면 전체를 활용하지 못합니다.

해결 방향: 비율 강제 로직 제거. 최소 크기(900×600)만 유지하고 자유 리사이즈 허용.
BLOCKER 잠재적 NameError — s_all 미초기화
kpi_view.py refresh()에서 for card, ms in zip(self._cards, [all_matches, today]) 루프 내에서 s_all = s를 조건부로 할당합니다. 카드 목록이 비어 있거나 루프가 0회 실행되면 self.overall_lbl.setText(...s_all...)에서 NameError가 발생합니다.

해결 방향: 루프 전에 s_all = stats.win_rate_summary([])로 기본값 초기화.
BLOCKER 피드백 URL — 하드코딩 없이 빈 값으로 배포 시 오류
feedback_form_url이 ocr_config.json에 없거나 비어 있으면 피드백 다이얼로그가 오류 메시지를 표시합니다. 배포 버전에서 이 URL을 코드에 직접 고정하거나, 설정 UI에서 입력받아야 합니다.
BLOCKER 설정 뷰에 업데이트 확인 없음
updater.py가 있지만 설정 화면에서 호출되지 않습니다. 일반 유저가 새 버전을 인지할 방법이 없습니다.

해결 방향: 설정 뷰 하단에 "업데이트 확인" 버튼 + 현재 버전 표시. 앱 시작 시 백그라운드 업데이트 체크.

⚠️ High 이슈 (첫 배포 전 처리 권장)

HIGH OCR 버튼 명칭 — "● OCR"은 일반 유저에게 불명확
"OCR"이 뭔지 모르는 유저가 대다수입니다. "● 자동 기록"으로 변경하고, 활성화 시 "⏹ 기록 중"으로 표시하는 것이 직관적입니다. 툴팁에 "마스터듀얼 화면을 자동으로 인식해 대전 결과를 기록합니다"를 추가하세요.
HIGH "전체 삭제" 버튼 — 복구 불가 동작의 UX 위험성
기록 탭의 "전체 삭제"와 덱 관리의 "전체 삭제"가 "선택 삭제" 바로 옆에 배치되어 있습니다. 확인 다이얼로그가 있지만 버튼 위치·색상 차이가 작아 실수 위험이 높습니다.

해결 방향: "전체 삭제"를 우측 끝 또는 별도 구역으로 분리. 빨간 배경 강조. 추가로 30초 실행 취소(Undo) 토스트 배너를 제공하면 완벽합니다.
HIGH 덱 이름 수정 불가 — 삭제 후 재등록 필요
덱 관리 탭에서 등록된 덱의 이름을 수정할 수 없습니다. 오탈자 수정 시 삭제 → 재등록해야 합니다. 이 경우 기존 대전 기록의 덱 이름이 변경되지 않아 통계가 분리됩니다.

해결 방향: 덱 이름 열 더블클릭 인라인 편집 + DB/기록 일괄 rename 기능.
HIGH 필터 초기화 버튼 없음
기록 탭과 대시보드 모두 필터바에 "초기화" 버튼이 없습니다. 필터를 걸었다가 취소하려면 각 콤보박스를 하나씩 "전체"로 되돌려야 합니다.
HIGH 매치업 카드 4열 고정 + 카드 크기 고정
_COLS = 4, _CARD_W = 160, _CARD_H = 96이 하드코딩되어 있어 창 크기 변경·고해상도 모니터에서 카드가 남은 공간을 채우지 못하거나 덱 이름이 긴 경우 잘립니다.

해결 방향: FlowLayout 또는 동적 열 계산으로 반응형 그리드 구현.
HIGH 추세 뷰 X축 — 날짜 간격이 균등하지 않아도 같은 간격으로 표시
xs = list(range(len(points)))로 정수 인덱스를 X축으로 사용합니다. 1일차와 30일차, 31일차가 모두 같은 간격으로 그려집니다. 플레이 안 한 날이 건너뛰어져서 추세가 왜곡됩니다.

해결 방향: X축을 날짜 기준 실제 간격(epoch 또는 ordinal)으로 변경.
HIGH 상태바 과밀 — 2줄 76px에 너무 많은 컨트롤
기록 탭 상태바에 OCR 버튼, 상태 텍스트, 오늘 요약, 내 덱, 상대 덱, 타입, + 버튼이 76px 안에 몰려 있습니다. 창이 작아지면 컨트롤이 눌리거나 레이블이 잘립니다.

해결 방향: 상태바를 "OCR 제어 영역"과 "세션 입력 영역"으로 시각적으로 분리. 내 덱/타입 선택은 사이드바나 별도 세션 패널로 이동 검토.
HIGH CSV 수출 → 내보내기 (표기 오류)
"수출"은 무역 용어. 소프트웨어에서의 자연스러운 한국어는 "내보내기 / 가져오기"입니다. 현재 덱 관리의 "엑셀 내보내기/가져오기"는 올바른 표현이지만, 기록 탭의 "CSV 수출"은 불일치합니다.
HIGH 미확정 행 하이라이트 — 유저에게 의미 불명확
어두운 갈색(#231c0a) 행이 표시되지만 왜 강조되는지 UI에 설명이 없습니다. 새 유저는 이것이 오류인지, 다른 타입인지 알 수 없습니다.

해결 방향: 테이블 상단에 "⚠ N건: 상대 덱 또는 결과를 입력해주세요" 배너를 표시. 미확정 행에 툴팁("클릭하여 수정") 추가.
HIGH 기록 탭과 대시보드에 필터바 중복 — 상태가 연동되지 않음
기록 탭 필터바와 대시보드 필터바가 별개 인스턴스입니다. 한쪽에서 필터를 걸어도 다른 탭엔 적용되지 않아 혼란을 줍니다.

해결 방향: 전역 필터 상태를 앱 레벨에서 단일 관리하거나, 기록 탭 필터를 대시보드 필터와 동기화.
HIGH 덱 관리 — 새 덱 추가 시 is_mine=False 기본값, 이후 체크해야 함
새 덱을 추가하면 기본으로 "내 덱 아님"으로 저장됩니다. 유저가 "+ 추가" 버튼을 눌렀다는 것은 본인 덱일 가능성이 높은데, 매번 테이블에서 체크박스를 다시 켜야 합니다.

해결 방향: 추가 폼에 "내 덱으로 등록" 체크박스를 기본 체크 상태로 포함.

🔵 Medium 이슈 (초기 배포 후 업데이트에서 처리)

MEDIUM KPI 카드 — 전체·오늘만 있음, 7일·30일·이번 달 없음
대시보드 KPI 탭과 기록 탭 미니 KPI 모두 "전체"와 "오늘"만 표시합니다. 랭크전 유저가 가장 많이 보고 싶어하는 "최근 7일", "이번 달" 통계가 없습니다.
MEDIUM 랭크 정보 — 필드는 있지만 UI 표시 없음
Match.rank_label 필드가 있고 수동 입력 다이얼로그에서 받지만, 대시보드에 랭크 이력이 전혀 표시되지 않습니다. 추세 뷰에 랭크 마커 오버레이 추가를 권장합니다.
MEDIUM 최장 연승/연패 기록 없음 — 현재 스트릭만 표시
stats.py는 현재 스트릭만 계산합니다. 유저가 가장 흥미로워하는 역대 최장 연승/연패는 계산되지 않습니다. KPI 카드에 "최고 연승 N연승" 추가를 권장합니다.
MEDIUM 실행 취소(Undo) 없음
삭제, 덱 편집, 인라인 수정 모두 즉시 DB에 반영되고 되돌릴 수 없습니다. 실수로 기록을 지우면 복구 불가합니다. 최소한 삭제 직후 5초간 "실행 취소" 토스트 배너를 제공하는 것을 권장합니다.
MEDIUM 툴팁 부족 — 대부분 버튼에 설명 없음
OCR 토글, 선택 삭제, 전체 삭제, 덱 관리 버튼 등에 툴팁이 없거나 불충분합니다. 처음 사용하는 유저가 각 버튼의 역할을 텍스트만으로 파악해야 합니다.
MEDIUM pyqtgraph 기본 테마 — 앱 다크 테마와 불일치
추세 뷰와 메타 뷰의 pyqtgraph 플롯이 배경색은 맞췄지만, 범례 배경이 하얗게 나오거나 축 스타일이 앱 전체 디자인과 어색하게 다를 수 있습니다. 플롯 내부 폰트도 앱 폰트(Noto Sans KR)와 다릅니다.
MEDIUM 메타 뷰 — 덱이 많아질수록 Y축 레이블 겹침
상대 덱 분포 막대 그래프에서 덱이 20개 이상이면 pyqtgraph Y축 레이블이 겹칩니다. 스크롤 가능한 레이아웃이나 상위 N개만 표시하는 필터가 필요합니다.
MEDIUM 버튼 스타일 불일치 — 일부 뷰가 테마 변수 미사용
deck_view.py의 버튼들이 #ef4444, #7f1d1d, #16a34a, #0369a1을 직접 하드코딩합니다. theme.py의 WIN, LOSE 상수와 불일치. 테마 변경 시 반영이 안 됩니다.
MEDIUM 데이터 백업/복원 없음
CSV 내보내기는 있지만, DB 전체 백업(SQLite 파일 복사)과 복원 기능이 없습니다. 실수 또는 앱 이전 시 데이터 손실 위험이 있습니다. 설정 뷰에 "데이터 백업" / "데이터 복원" 버튼 추가를 권장합니다.
MEDIUM 테이블 인라인 편집 — 편집 가능 열이 불명확
기록 테이블에서 "내 덱"과 "상대 덱"만 더블클릭 편집이 가능하지만, 시각적으로 다른 열과 구분이 없어서 유저가 어떤 열을 편집할 수 있는지 알기 어렵습니다. 편집 가능한 열에 연필 아이콘 또는 밑줄 등 시각 힌트가 필요합니다.

📊 통계 개선 제안

LOW 무승부 처리 방식 — UI에 명시 없음
무승부가 승률 분모에서 제외된다는 설계가 UI에 전혀 표시되지 않습니다. 유저가 "10전 5승 2패 3무인데 왜 승률이 71.4%지?" 하는 혼란이 생깁니다. KPI 카드나 승률 옆에 "(무승부 제외)" 소주석을 추가하세요.
LOW 시즌별 통계 분리 — season 필드 있지만 집계 없음
Match.season이 YYYY-MM 형식으로 자동 기록되지만 대시보드 필터나 통계 함수에서 시즌별 집계를 제공하지 않습니다. filter_bar에 시즌(월) 필터 추가를 권장합니다.
LOW 일평균 게임 수, 세션 길이 등 플레이 패턴 통계 없음
순수 승률 외에 "하루 평균 N게임", "이번 달 총 플레이 시간 추정", "가장 많이 플레이한 요일" 등의 통계가 없습니다. 추세 뷰에 일별 게임 수 바 차트를 추가하면 유저 참여도가 올라갑니다.
LOW 매치업 뷰 — 표본 작은 카드의 신뢰도 경고 없음
상대 덱을 1~2회만 만났어도 승률이 100%/0%로 크게 표시됩니다. 표본이 3 미만인 카드에 "⚠ 표본 부족" 오버레이를 표시하거나 카드를 흐리게 표시하는 것을 권장합니다.
